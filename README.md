# FintechScreener (Console Core backend API)

Консольное бэкенд-приложение, симулирующее ядро банковской системы (P2P-переводы, фрод-мониторинг, система лояльности). Проект спроектирован с упором на отказоустойчивость, системную логику и чистоту управления памятью.

## 🛠 Технологический стек
* **Платформа:** .NET 8 (C#)
* **СУБД:** SQLite (внедрена embedded-интеграция для локального тестирования без развертывания тяжелой инфраструктуры)
* **ORM:** Entity Framework Core

## 🏛 Архитектура проекта
Приложение построено по классической **трехслойной архитектуре**:
1. **Слой данных (Data/Entities):** Описание таблиц `User` и `Transaction` со связью один-ко-многим. Настройка контрактов точности (`decimal(18,2)`) для предотвращения потери копеек.
2. **Слой бизнес-логики (Services):** Слабосвязанная архитектура через интерфейсы (`ITransactionService`).
3. **Точка входа (Program.cs):** Имитация работы API-контроллеров и интеграционное тестирование сценариев сбоя.

## 🚀 Ключевые фичи и финтех-оптимизации
* **Атомарные P2P-переводы:** Использование механизмов `BeginTransactionAsync()`, `CommitAsync()` и `RollbackAsync()` для защиты балансов клиентов при сбоях на сервере.
* **Fail-Fast Валидация:** Проверки на уровне ядра (защита от отрицательных сумм перевода, переводов самому себе, ограничение пагинации).
* **Фрод-мониторинг:** Метод `IsSuspiciousActivityAsync` выявляет подозрительную активность (более 3 транзакций за 5 минут). Оптимизирован через LINQ-метод `.CountAsync()` без выгрузки данных в память.
* **Система лояльности:** Начисление 1% кешбэка в отдельную копилку с применением **банковского округления** (`Math.Round(x, 2)`) до копеек.
* **Управление памятью:** Применение `context.ChangeTracker.Clear()` при краш-тестах для очистки кэша оперативной памяти C# и синхронизации объектов с жестким диском.
